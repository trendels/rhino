Adding ETags

When you include an ETag header with a response, Rhino will automatically handle conditional GET and HEAD requests for you. The easiest way to add an ETag is to use the etag parameter of the response helper functions like rhino.response.ok (see here for the complete list).

When using the etag parameter, quotes will be added to the etag value automatically.

@get
def hello(request):
    body = "Hello, world!"
    return ok(body, etag="hello-1")

The above will result in a response like this being sent:

HTTP/1.1 200 OK
Etag: "hello-1"
...

Hello, world!

A browser will typically save the ETag and send it with subsequent requests to the same URL as the value of the If-None-Match request header. If the two ETags match, Rhino will send back a 304 Not Modified response without a response body, which tells the browser that the page has not changed, and it can use its locally cached version. This can result in substantial bandwidth savings.

Automatically computed ETags

The value of the ETag must change when the representation changes in order to prevent stale responses being cached. The etag argument also accepts a callable, which will be called with the response body as argument to produce the ETag header value. You use this to send ETags that change automatically whenever the response body changes:

from hashlib import md5

def md5_etag(body):
    return md5(body.encode('utf-8')).hexdigest()

@get
def hello(request):
    body = "Hello, world!"
    return ok(body, etag=md5_etag)

This will result in a response like this being sent:

HTTP/1.1 200 OK
Etag: "6cd3556deb0da54bca060b4c39479839"
...

Hello, world!

“Deep” ETags

The solution above will still create the response body for every request. For resources where this is a computationally expensive process, you may want to precompute the ETag headers, and store them in a cache, so you can handle conditional requests without having to produce the response body unless it is actually needed. For this reason, Rhino accepts a callable as the response body, which will only be called for non-304 responses:

@get
def expensive_resource(request):
    etag = cheap_lookup()
    def lazy_response_body():
        return expensive_calculation()
    return ok(lazy_response_body, etag=etag)

Other caching-related headers

Last-Modified

In addition to ETags, conditional requests can also be handled based on ‘Last-Modified’ headers. See rhino.response.response for the supported formats. However, ETags are usually preferred since the HTTP date format only provides 1-second precision.

Cache-Control and Expires

These headers are different in that they are not used to validate the “freshness” of a response, like ETag and Last-Modified, but instruct the if or for how long it is allowed to cache responses without checking back with the origin server. See rhino.response.response on the supported formats for the expires header. The rhino.http.cache_control helper function can be used to generate Cache-Control headers:

from rhino import cache_control as cc

@get
def cached_resource(request):
    body = "Hello, world!"
    return ok(body, cache_control=cc(public=True, max_age=3600))
